API contract

Request

GET /api/snapshot
  ?days=365                   // optional, default 365, max 365

Response — Snapshot

{
  generated_at: string,       // ISO
  tz: string,                 // 'America/New_York'
  channels: {
    github: {
      days: Array<{ date: 'YYYY-MM-DD', count: number }>,
      streak_current: number,
      streak_longest: number,
      today_count: number
    },
    twitter: { /* same shape */ }
  },
  combined: { streak_current, streak_longest, mode: 'or' | 'and' }
}

Algorithm — streak

  1. Build sorted days[] from daily_count for channel, filling missing dates with 0.
  2. ship[i] = days[i].count >= threshold (threshold default 1).
  3. Current streak: walk backward from today. If !ship[today] and now < cutoff_time, treat today as pending and start from yesterday; else streak ends at today.
  4. Increment counter while ship[i]; stop at first false.
  5. Longest: single pass, max consecutive true.
  6. Combined: ship_combined[i] = ship_gh[i] OP ship_tw[i] where OP ∈ {OR, AND} per Q-004.

UI layout

  1. <TopBar/> — NERV hex logo, title CENTRAL DOGMA, live clock, sync indicator.
  2. <MagiPanel channel="github"/> — left. Streak (big), today count, 53×7 heatmap.
  3. <MagiPanel channel="twitter"/> — right. Same shape.
  4. <StatusBar/> — bottom. SYS:OK, last sync, combined streak, rotating MAGI flavor text.
  5. <ScanlineOverlay/> — fixed, pointer-events none, 1px scanlines @ 6% alpha + vignette.
  6. <RedAlert/> — full-screen takeover when !ship_today_combined AND now >= alert_time.

Decisions

IDChoiceRationale
D-001Stack = Next.js + Tailwind + better-sqlite3.SSR makes initial paint instant; SQLite is one file, zero ops; Tailwind makes the NERV skin tokens trivial.
D-002GitHub source = GraphQL contributionsCollection.Returns exact data shown on github.com profile, one call, includes private if token has read:user.
D-003Streaks computed at read-time from daily_count.365 rows; trivial cost; eliminates denormalization bugs.
D-004Always wire the ship self-report CLI in addition to any API-backed Twitter source.X APIs/scrapers break; manual log is the floor that keeps the streak honest.
D-005Pluggable twitter adapter interface.Whichever Q-001 answer, swapping later is one file.
D-006Local-first deploy via pnpm dev + launchd cron; Tauri later.Ship the dashboard this week; Tauri is a wrapper, not a blocker. Superseded by D-008.
D-007Twitter ingest via pi-chrome bridge driving user's signed-in Chrome.Bypasses the $100/mo X API; uses already-authenticated profile; pi-chrome already installed locally. Resolves Q-001.
D-008Deploy = Vercel (frontend) + Turso libSQL (DB). Ingestor stays local on Mac. Supersedes D-006.User wants phone access. Ingestor cannot move (needs Chrome). Turso bridges local writes to hosted reads with the same SQL.
D-009Two-tier Twitter scraping: deterministic DOM via chrome_evaluate first; LLM extraction from chrome_snapshot only on failure.Deterministic is free and fast; LLM is robust to X DOM changes but costs cents/call. Tier-2 only fires when tier-1 returns 0 twice in a row.
D-010Phase-2 invocation of pi-chrome = shell out to pi --prompt <script> --json.Zero plumbing; lets pi-chrome handle auth + tool dispatch. Move to direct bridge HTTP calls (port 17318) in Phase 5 only if per-call latency matters.
D-011Combined streak mode = AND. TZ = America/Los_Angeles. Resolves Q-004, Q-005.User explicit. Harder mode = more accountability.

Tradeoffs

IDPicked over RejectedReason
T-001Next.js web app over Tauri desktop app for v1.Browser tab ships in days, native window is polish. Tauri can wrap the same web app later (L-001).
T-002SQLite over JSON file.Proper indexed reads, transactions, room for the event table without inventing a format.
T-003GraphQL contributionsCollection over scraping github.com/<user> profile SVG.Official, stable shape, includes private contribs with a token.
T-004Re-fetch full 365-day window each ingest over incremental.One small call; idempotent upsert; no edge cases on backfill.
T-005OR-combined streak default over AND.AND punishes you for shipping code on a heads-down day; OR rewards any output. Configurable (Q-004).
T-006Pluggable twitter adapter over hardcoding one source.X access keeps changing; isolating the failure point is cheap.
T-007Turso (libSQL) over Supabase (Postgres) or local-only SQLite.Same SQL as our local dev file (zero migration cost); free tier ample; libSQL HTTP client works in Vercel edge runtime.
T-008Vercel frontend + local Mac ingestor over fully-local Tauri app.Phone access requirement. Vercel is free for one user. Tauri stays on L-001 if we later want a desktop widget.
T-009Shell-out to pi --prompt over directly hitting bridge HTTP :17318.Bridge protocol is undocumented and would need reverse-engineering from the extension source. Shell-out works today.
T-010AND-combined streak over OR.User picked harder mode. Reverses T-005 default for this user.

Assumptions

  • A-001 — User runs macOS; launchd is available for cron.
  • A-002 — Node 20+ and pnpm available.
  • A-003 — Counting contributions (commits + PRs + issues + reviews, GitHub's own definition) is the right metric, not raw commits only.
  • A-004 — One GitHub identity (anishthite), one X identity. No multi-account.
  • A-005 — Tweets count regardless of whether they are replies or originals. If wrong, filter in scrape.js.
  • A-006 — User has pi-chrome onboarded (/chrome doctor green) with their primary Chrome profile signed into x.com.
  • A-007 — User's Mac is awake and Chrome is running at scheduled ingest ticks. Missed ticks tolerated — next tick re-scans full window.
  • A-008 — Twitter scrape window = last 30 days per tick (GitHub window = 365). Old days locked in.
  • A-009 — One-shot pi --prompt can authorize Chrome non-interactively after initial /chrome authorize indefinite. Verify in Phase 2; if false, fall back to a long-lived background Pi session.

Reversibility

R-001 — Entire project is greenfield. Local ingestor only reads GitHub + reads x.com via your own session; writes only to Turso. Revert = rm -rf + delete Turso DB + delete Vercel project. Blast: low.

R-002 — pi-chrome bridge is loopback-only and refuses browser-origin requests; revoke any time with /chrome revoke. Blast: low.
